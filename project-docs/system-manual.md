# DKH 업무관리시스템 상세 운영 매뉴얼

작성일: 2026-05-14

이 문서는 단순 화면 설명이 아니라, 실제 사용 중 헷갈리기 쉬운 자동 입력, 저장 조건, 예외 상황, 집계 제외 기준, 수주대장 연동, 월마감/일일판매 반영 로직을 함께 정리한 상세 운영 매뉴얼입니다.

## 1. 기본 개념

### 1.1 주요 데이터 흐름

시스템의 중심 흐름은 다음과 같습니다.

1. `납품처 관리`, `공급자 관리`, `품목 관리`에서 기준 정보를 등록합니다.
2. `문서 작성`에서 납품처, 수신처, 품목을 선택해 출고의뢰서/거래명세서를 작성합니다.
3. 문서를 저장하면 `발행 이력`에 문서가 저장되고, 각 품목은 `수주대장`에 자동 생성됩니다.
4. 수주대장에서 품목별 `미출고/출고` 상태를 관리합니다.
5. 발행 이력과 품목 상태를 기준으로 `월마감`, `일일판매`, `대시보드`가 집계됩니다.

### 1.2 상태 코드

| 코드 | 화면 표시 의미 | 적용 대상 | 집계 영향 |
| --- | --- | --- | --- |
| `ST00` | 정상, 진행중 | 문서, 문서 품목, 수주대장 | 집계 포함 |
| `ST01` | 거래취소 | 문서, 문서 품목, 수주대장 | 월마감/일일판매/대시보드 집계에서 제외 |

### 1.3 삭제 기준

화면에서 삭제한 데이터는 대부분 실제 DB 행을 바로 지우지 않고 `del_yn = 'Y'`로 소프트 삭제합니다.

- 일반 조회는 `del_yn = 'N'`만 보여줍니다.
- 삭제된 납품처, 품목, 문서, 수주 항목은 일반 목록과 집계에서 제외됩니다.
- 운영상 이미 문서에 사용된 마스터 데이터는 삭제보다 비활성 또는 새 품목 생성 방식이 더 안전합니다.

## 2. 로그인과 권한

### 2.1 로그인

로그인 화면에서 아이디와 비밀번호를 입력합니다.

로그인 실패 상황:

- 아이디 또는 비밀번호가 비어 있으면 로그인하지 않고 안내 메시지를 표시합니다.
- 계정이 없거나 비밀번호가 맞지 않으면 `아이디 또는 비밀번호가 올바르지 않습니다.`가 표시됩니다.
- 잠긴 계정이면 `잠긴 계정입니다. 관리자에게 문의해 주세요.`가 표시됩니다.
- 로그인 실패가 누적되면 계정 잠금 처리될 수 있습니다.

### 2.2 세션

- 로그인 성공 시 사용자 정보가 브라우저 저장소에 저장됩니다.
- 저장되는 정보는 아이디, 이름, 직급, 권한입니다.
- 로그인 정보가 없으면 보호된 화면 접근 시 로그인 화면으로 이동합니다.
- 로그아웃하면 저장된 사용자 정보가 삭제됩니다.

### 2.3 권한별 메뉴

| 권한 | 접근 가능 메뉴 |
| --- | --- |
| 일반 사용자 | 대시보드, 문서 작성, 발행 이력, 수주대장, 납품처 관리, 품목 관리, 공급자 관리, 월마감, 보도자료 |
| 관리자 | 일반 사용자 메뉴 전체, 일일판매, 계정 관리 |

관리자 전용 메뉴에 일반 사용자가 직접 접근하면 권한 없음 안내 후 대시보드로 이동합니다.

## 3. 대시보드

대시보드는 수주대장의 입고 예정 항목을 기준으로 현황을 보여줍니다.

### 3.1 오늘의 할 일

오늘 날짜 기준 입고 예정 항목을 보여줍니다.

집계 기준:

- 수주대장 항목 중 삭제되지 않은 항목만 대상입니다.
- 거래취소 상태(`ST01`)는 별도로 거래취소 건수로 분류됩니다.
- 출고상태가 `미출고`인 항목은 미출고 건수로 잡힙니다.

### 3.2 지연 건수

입고일이 오늘보다 이전인데 아직 `미출고` 상태인 항목을 보여줍니다.

주의:

- 거래취소된 항목은 지연 건수의 실제 처리 대상에서 제외해야 합니다.
- 입고일이 비어 있으면 날짜 기준 지연 여부를 판단하기 어렵습니다.

### 3.3 입고예정건수

주간 단위로 입고 예정 건수를 보여줍니다.

사용 방식:

- 이전 주, 다음 주 버튼으로 주간 범위를 이동합니다.
- 날짜 버튼을 클릭하면 해당 날짜의 상세 항목 패널을 확인할 수 있습니다.
- 패널에서 출고상태를 `미출고` 또는 `출고`로 바꿀 수 있습니다.

### 3.4 최근 등록 문서

최근 작성 또는 수정된 문서를 보여줍니다.

- 행을 클릭하면 발행 이력 상세로 이동합니다.
- 거래취소 문서는 상태에 따라 표시가 달라질 수 있습니다.

## 4. 문서 작성

문서 작성은 출고의뢰서와 거래명세서를 동시에 만들기 위한 화면입니다.

## 4.1 기본 정보 입력 로직

### 발급번호

- 화면 진입 시 기존 문서의 가장 최근 발급번호를 조회합니다.
- 최근 발급번호를 숫자로 해석할 수 있으면 `+1`한 값을 기본 발급번호로 사용합니다.
- 최근 발급번호가 없거나 숫자로 해석할 수 없으면 `26001`을 사용합니다.
- 발급번호는 화면에서 직접 수정할 수 있습니다.

예외:

- 발급번호를 직접 바꾸면 별도 자동 검증 없이 저장 payload에 반영됩니다.
- 발급번호 변경 사유는 `발급번호 수정이력`에 직접 남겨야 추적이 쉽습니다.

### 납품처

납품처는 반드시 목록에서 선택해야 합니다.

자동 입력 항목:

- 담당자
- 담당자 연락처
- 납품주소
- 유의사항

유의사항 자동 구성:

- 납품처의 `입고시간`이 있으면 `입고시간 : ...` 형식으로 들어갑니다.
- 납품처의 `점심시간`이 있으면 `점심시간 : ...` 형식으로 들어갑니다.
- 납품처의 비고가 있으면 함께 붙습니다.
- 여러 값은 ` / `로 이어집니다.

예외:

- 납품처명을 직접 타이핑만 하고 목록에서 선택하지 않으면 `clientId`가 없기 때문에 저장할 수 없습니다.
- 납품처를 변경하면 이미 선택된 등록 품목은 초기화됩니다.
- 직접입력 품목은 납품처 변경 후에도 내용이 남을 수 있으나, 실제 업무상 새 납품처에 맞는 품목인지 확인해야 합니다.

### 수신처

- 수신처는 목록에서 선택하거나 직접 입력할 수 있습니다.
- 납품처와 수신처가 모두 있어야 품목 목록이 표시됩니다.
- 수신처를 변경하면 기존 등록 품목 선택값이 초기화됩니다.

## 4.2 품목 입력 로직

### 품목 선택 가능 조건

품목 선택 목록은 다음 조건을 만족해야 표시됩니다.

- 납품처가 목록에서 선택되어 `clientId`가 있어야 합니다.
- 수신처가 입력되어 있어야 합니다.
- 해당 납품처의 품목 중 수신처가 같은 품목만 보여줍니다.

예외:

- 조건을 만족하지 않으면 `납품처와 수신처를 모두 선택해야 품목 리스트가 표시됩니다.` 메시지가 표시됩니다.
- 그래도 `직접입력` 품목은 사용할 수 있습니다.

### 등록 품목 선택 시 자동 반영

등록된 품목을 선택하면 다음 값이 품목 기준 정보에서 들어옵니다.

- 품목명(출고의뢰서)
- 품목명(거래명세서)
- 구분
- 입고단가
- 판매단가
- 1B=EA
- 1P=BOX
- 1P=EA
- 1대당 파레트

### 직접입력 품목

직접입력을 선택하면 사용자가 품목명과 구분을 입력합니다.

직접입력 품목의 특징:

- `productId`는 저장되지 않습니다.
- 품목명(출고의뢰서)과 품목명(거래명세서)은 같은 값으로 저장됩니다.
- 포장 기준이 없으므로 파레트/BOX는 자동 계산되지 않습니다.
- 파레트/BOX가 필요하면 직접 입력합니다.

### 수량, 단가, 공급가액 계산

계산 기준:

- 판매단가가 비어 있으면 기본적으로 `0`으로 계산합니다.
- 공급가액 기본값은 `판매단가 × 수량`을 반올림한 값입니다.
- 공급가액을 직접 입력하면 자동 계산값보다 직접 입력값이 우선합니다.
- 판매단가를 수정하면 직접 입력한 공급가액은 초기화되고 다시 자동 계산됩니다.
- 수량을 수정하면 파레트, BOX, 공급가액 직접 입력값이 초기화됩니다.

### VAT 계산

- VAT 체크가 켜져 있으면 `공급가액 × 10%`를 반올림합니다.
- VAT 체크가 꺼져 있으면 부가세는 `0`입니다.
- 합계금액은 `공급가액 + 부가세`입니다.

### 파레트/BOX 계산

자동 계산 기준:

- BOX = `수량 ÷ 1B=EA`를 올림 처리합니다.
- 파레트 = `수량 ÷ (1B=EA × 1P=BOX)`를 올림 처리합니다.
- 수량이 0 이하이거나 기준값이 없으면 자동 계산하지 않습니다.

예외:

- 파레트 또는 BOX를 직접 입력하면 직접 입력값이 우선합니다.
- 수량을 다시 바꾸면 직접 입력한 파레트/BOX가 초기화됩니다.
- 직접입력 품목은 기준값이 없으므로 자동 계산되지 않습니다.

### 품목 삭제

- 품목 행이 1개만 남아 있으면 삭제되지 않습니다.
- 2개 이상일 때만 해당 행을 목록에서 제거합니다.
- 신규 작성 화면에서는 품목 삭제가 DB에 반영되는 것이 아니라 저장 전 화면 상태에서만 제거됩니다.

## 4.3 저장 조건

문서 저장 전 필수 검증:

| 조건 | 실패 메시지 |
| --- | --- |
| 발주일 없음 | 발주일을 입력해 주세요. |
| 입고일 없음 | 입고일을 입력해 주세요. |
| 납품처명 없음 | 납품처를 선택해 주세요. |
| 납품처 ID 없음 | 납품처를 목록에서 선택해 주세요. |
| 수신처 없음 | 수신처를 입력해 주세요. |
| 납품주소 없음 | 납품주소를 입력해 주세요. |
| 유효 품목 없음 | 저장할 품목을 1개 이상 입력해 주세요. |

유효 품목 기준:

- 품목명(`name1`)이 있어야 합니다.
- 등록 품목이면 선택한 품목의 품목명이 있어야 합니다.
- 직접입력이면 직접 입력한 품목명이 있어야 합니다.

중복 저장 방지:

- 저장 중에는 내부 저장 잠금이 걸립니다.
- 이미 저장 중이면 다시 저장하지 않습니다.
- API 레벨에서도 `문서 저장이 이미 진행 중입니다. 잠시 후 다시 시도해 주세요.` 예외가 있습니다.

## 4.4 저장 시 DB 반영 로직

저장 버튼을 누르면 다음 순서로 처리됩니다.

1. 문서 헤더가 `documents`에 저장됩니다.
2. 품목 목록이 `document_items`에 저장됩니다.
3. 저장된 각 품목마다 `order_book`에 수주 항목이 생성됩니다.
4. 저장 후 발행 이력 상세 화면으로 이동합니다.

수주대장 자동 생성 기준:

- `doc_id`: 저장된 문서 ID
- `document_item_id`: 저장된 문서 품목 ID
- `issue_no`: 문서 발급번호
- `client_id`: 납품처 ID
- `product_id`: 등록 품목이면 품목 ID, 직접입력이면 비어 있음
- `date`: 문서 발주일
- `deadline`: 품목 입고일이 있으면 품목 입고일, 없으면 문서 입고일
- `client`: 납품처명
- `product`: 품목명(출고의뢰서)
- `qty`: 수량
- `note`: 문서 유의사항
- `status`: 문서 상태
- `shipped_status`: 기본 `미출고`
- `from_doc`: `true`

예외:

- 문서 헤더 저장 후 품목 또는 수주대장 저장에서 오류가 나면, 생성된 문서 헤더를 삭제하려고 시도합니다.
- 이 경우 사용자에게 저장 실패 오류가 표시될 수 있습니다.

## 4.5 미리보기와 엑셀 다운로드

미리보기와 엑셀 다운로드는 저장과 거의 같은 필수 조건을 확인합니다.

다만 `저장할 품목을 1개 이상 입력해 주세요.` 조건은 별도로 처리됩니다.

- 기본 정보가 부족하면 먼저 해당 필수값 안내가 뜹니다.
- 품목이 없으면 `미리보기 전에 품목을 먼저 입력해 주세요.` 또는 `엑셀 다운로드 전에 품목을 먼저 입력해 주세요.`가 표시됩니다.

## 4.6 발행이력 불러오기

기존 문서를 불러오면 다음 정보가 현재 작성 화면에 채워집니다.

- 발주일, 입고일
- 납품처, 담당자, 연락처
- 수신처, 납품주소
- 유의사항, 요청사항
- 발급번호 수정이력
- 공급자 정보
- 품목 정보

중요:

- 불러온 문서의 발급번호를 그대로 쓰면 중복 발급번호가 될 수 있습니다.
- 새 문서로 저장할 경우 발급번호와 날짜를 반드시 확인합니다.
- 불러온 품목의 기존 `productId`가 현재 납품처 품목 목록에 없으면 등록 품목 선택이 풀릴 수 있습니다.

## 5. 발행 이력

발행 이력은 저장된 문서를 조회하고 수정하는 화면입니다.

## 5.1 목록 조회 로직

조회 조건:

- 삭제되지 않은 문서(`del_yn = 'N'`)만 조회합니다.
- 날짜 조건은 `arrive_date`, 즉 문서 입고일 기준입니다.
- 검색 필터는 전체, 거래처, 수신처, 작성자입니다.

전체 검색 대상:

- 발급번호
- 거래처
- 수신처
- 작성자

예외:

- 품목명 검색은 발행 이력 목록 필터에 직접 포함되어 있지 않습니다.
- 품목명으로 찾으려면 수주대장 또는 문서 작성의 불러오기 검색이 더 적합합니다.

## 5.2 상세 진입과 수정

문서 목록 행을 클릭하면 `/doc-history/{documentId}` 상세로 이동합니다.

상세에서 수정 가능한 항목:

- 발주일, 입고일
- 발급번호
- 거래처, 담당자, 연락처
- 수신처
- 납품주소
- 유의사항, 요청사항
- 발급번호 수정이력
- 공급자 정보
- 품목 정보

### 납품처/수신처 변경 시 품목 처리

상세 화면에서 납품처나 수신처를 바꾸면 등록 품목 선택이 초기화됩니다.

예외:

- 기존 문서를 열었을 때 납품처/수신처를 바꾸지 않은 상태라면 기존 품목 목록을 유지합니다.
- 납품처/수신처를 바꾼 경우에는 새 조건에 맞는 품목만 다시 선택할 수 있습니다.
- 납품처를 직접 타이핑만 하고 목록에서 선택하지 않으면 저장할 수 없습니다.

## 5.3 수정 저장 로직

수정 저장 시 처리 순서:

1. 문서 헤더를 업데이트합니다.
2. 화면에 남아 있는 품목을 기준으로 순번을 다시 매깁니다.
3. DB에는 있지만 화면에서 삭제된 품목은 `del_yn = 'Y'`로 처리합니다.
4. 기존 품목은 업데이트합니다.
5. 새로 추가한 품목은 새 행으로 insert합니다.
6. 문서에 연결된 수주대장을 품목 기준으로 다시 동기화합니다.

수주대장 동기화 기준:

- 문서 품목이 유지되면 기존 수주대장 행을 업데이트합니다.
- 문서 품목이 새로 생기면 수주대장 행도 새로 만듭니다.
- 문서 품목이 삭제되면 연결된 수주대장 행도 소프트 삭제합니다.
- 기존 수주대장의 `receipt`와 `shipped_status`는 최대한 유지합니다.
- 문서가 거래취소 상태이면 수주대장 상태도 `ST01`로 맞춥니다.

예외:

- 같은 문서를 동시에 저장하려고 하면 `같은 문서의 수정 저장이 이미 진행 중입니다. 잠시 후 다시 시도해 주세요.`가 발생합니다.
- 기존 수주대장 데이터에 `document_item_id`가 없는 과거 데이터가 있으면 안전하게 수정하지 않습니다.
- 이 경우 `기존 수주대장 데이터에 품목 연결 정보가 없어 안전하게 수정할 수 없습니다. 먼저 order_book document_item_id 백필 SQL을 실행해 주세요.`가 표시됩니다.

## 5.4 문서 전체 거래취소

문서 전체 거래취소를 누르면:

- `documents.status`가 `ST01`로 바뀝니다.
- 연결된 `order_book.status`도 모두 `ST01`로 바뀝니다.
- 월마감과 일일판매 집계에서 해당 문서 전체가 제외됩니다.

취소 해제를 누르면:

- 문서 상태가 `ST00`으로 돌아갑니다.
- 수주대장 상태는 각 문서 품목 상태에 맞춰 다시 동기화됩니다.
- 품목별로 취소된 항목은 계속 `ST01`로 남습니다.

예외:

- 이미 문서 상태 변경이 진행 중이면 `문서 상태 변경이 이미 진행 중입니다. 잠시 후 다시 시도해 주세요.`가 발생합니다.
- 문서가 삭제되었거나 권한 문제가 있으면 상태 변경 실패 메시지가 표시됩니다.

## 5.5 품목별 거래취소

품목취소를 누르면:

- 해당 `document_items.status`가 `ST01`로 바뀝니다.
- 해당 품목과 연결된 `order_book.status`도 바뀝니다.
- 문서 합계가 취소되지 않은 품목 기준으로 다시 계산됩니다.

품목취소 해제:

- 품목 상태가 `ST00`으로 돌아갑니다.
- 단, 문서 전체가 거래취소 상태이면 수주대장 상태는 문서 상태를 우선하여 `ST01`로 유지됩니다.

예외:

- 새로 추가했지만 아직 저장하지 않은 품목은 취소 처리할 수 없습니다.
- 화면에서는 `새로 추가한 품목은 먼저 저장한 뒤 취소 처리할 수 있습니다.`가 표시됩니다.
- 이미 같은 품목의 상태 변경이 진행 중이면 `품목 상태 변경이 이미 진행 중입니다. 잠시 후 다시 시도해 주세요.`가 발생합니다.

## 6. 수주대장

수주대장은 문서 품목 단위로 자동 생성되는 출고 관리 화면입니다.

## 6.1 조회 로직

조회 기준:

- 삭제되지 않은 수주 항목만 조회합니다.
- 날짜 조건은 `deadline`, 즉 입고일 기준입니다.
- 출고상태 필터는 전체, 미출고, 출고입니다.

검색 필터:

| 필터 | 검색 대상 |
| --- | --- |
| 전체 | 발급번호, 납품처, 상태/접수, 품목명, 비고, 수신처 |
| 납품처 | 수주대장 납품처 |
| 수신처 | 연결 문서의 수신처 |
| 품목명 | 수주대장 품목명 |
| 발급번호 | 발급번호 |
| 상태 | 접수/처리 상태 또는 상태 코드 |

수신처 검색 예외:

- 수주대장 테이블 자체에는 수신처가 직접 없고, 문서에서 찾아옵니다.
- 수신처 검색 시 먼저 문서에서 수신처가 맞는 문서 ID를 찾고, 그 문서 ID에 해당하는 수주대장을 조회합니다.

## 6.2 화면 표시 로직

수주대장 행은 문서와 문서 품목 정보를 다시 조합해서 표시합니다.

우선순위:

- 발급번호: 수주대장 값이 있으면 사용, 없으면 문서 발급번호 사용
- 발주일자: 연결 문서의 발주일 우선
- 입고일자: 문서 품목 입고일 우선, 없으면 문서 입고일, 없으면 수주대장 입고일
- 수신처: 연결 문서의 수신처
- 품목명: 문서 품목의 거래명세서명 또는 출고의뢰서명 우선, 없으면 수주대장 품목명
- 수량: 수주대장 수량 우선, 없으면 문서 품목 수량
- 파레트/BOX: 문서 품목의 직접 입력값 우선, 없으면 기준값으로 자동 계산

상태 우선순위:

- 연결 문서가 거래취소이면 수주대장 행도 거래취소로 표시합니다.
- 문서가 정상이라면 문서 품목 상태를 우선합니다.
- 문서 품목 상태가 없으면 수주대장 자체 상태를 사용합니다.

## 6.3 출고상태 변경

출고상태는 `미출고` 또는 `출고`입니다.

처리 방식:

- 행의 출고상태 드롭다운에서 바로 변경할 수 있습니다.
- 여러 행을 체크한 뒤 `일괄 출고처리`를 누르면 선택된 항목이 모두 `출고`로 변경됩니다.

예외:

- 아무 항목도 선택하지 않으면 일괄 출고처리는 실행되지 않습니다.
- 거래취소 항목도 화면에서 선택될 수 있으므로, 실제 운영에서는 취소 항목을 출고처리하지 않도록 확인해야 합니다.

## 6.4 수주 항목 수정

행을 클릭하면 수주 항목 수정 모달이 열립니다.

수정 가능 항목:

- 발급번호
- 수량
- 발주일자
- 입고일자
- 납품처
- 품목명
- 상태
- 출고상태
- 비고

문서 연동 항목:

- `from_doc = true`인 항목은 문서에서 생성된 항목입니다.
- 문서 연동 항목은 삭제 버튼이 표시되지 않습니다.
- 문서에서 다시 수정 저장하면 일부 값이 문서 기준으로 다시 동기화될 수 있습니다.

수동 등록 항목:

- `from_doc = false`인 항목은 수동 등록 항목입니다.
- 삭제 버튼으로 소프트 삭제할 수 있습니다.

## 7. 납품처 관리

납품처는 문서 작성의 가장 중요한 기준 데이터입니다.

## 7.1 납품처 등록/수정

필수 입력:

- 납품처명

선택 입력:

- 담당자
- 담당자 연락처
- 납품주소
- 입고시간
- 점심시간
- 비고
- 사용 여부

저장 전 처리:

- 입력값은 앞뒤 공백을 제거한 뒤 저장합니다.
- 사용중 체크가 꺼져 있으면 `active = false`로 저장됩니다.

## 7.2 비활성 납품처

- 문서 작성 화면에서는 `active !== false`인 납품처만 표시합니다.
- 기존 문서에 이미 사용된 납품처는 발행 이력에는 남아 있습니다.
- 운영상 거래가 끝난 납품처는 삭제보다 비활성이 안전합니다.

## 7.3 삭제

- 삭제 버튼을 누르면 `del_yn = 'Y'`로 바뀝니다.
- 삭제된 납품처는 목록과 신규 문서 작성 선택 목록에서 제외됩니다.

예외:

- 이미 문서나 품목에 연결된 납품처를 삭제하면 과거 데이터 추적이 어려워질 수 있습니다.
- 시스템은 삭제 전 연결 문서 존재 여부를 화면에서 강하게 검증하지 않으므로 운영자가 확인해야 합니다.

## 8. 품목 관리

품목 관리는 `공통 품목`과 `납품처별 품목`으로 나뉩니다.

## 8.1 공통 품목

공통 품목은 기준 품목입니다.

필수 입력:

- 품목명(출고의뢰서)

자동/기본 처리:

- 품목명(거래명세서)을 비워두면 출고의뢰서 품목명과 같은 값으로 저장합니다.
- 구분이 비어 있으면 기본 구분값으로 저장합니다.
- `1B=EA`와 `1P=BOX`가 모두 있으면 `1P=EA`는 두 값을 곱해서 계산합니다.
- 둘 중 하나가 없으면 입력된 `1P=EA` 값을 사용합니다.

삭제 예외:

- 공통 품목을 삭제하면 연결된 납품처별 품목도 함께 소프트 삭제됩니다.
- 삭제 전 확인창에 연결된 하위 품목 개수가 표시됩니다.

## 8.2 납품처별 품목

납품처별 품목은 실제 문서 작성에서 선택되는 품목입니다.

필수 입력:

- 공통 품목
- 납품처
- 품목명(출고의뢰서)

자동/기본 처리:

- 공통 품목을 선택하면 품목명, 구분, 포장 기준이 기본 적용될 수 있습니다.
- 납품처는 목록에서 선택되어 `clientId`가 있어야 합니다.
- 수신처는 문서 작성 시 품목 필터링에 사용됩니다.
- 품목명(거래명세서)을 비워두면 출고의뢰서 품목명과 같은 값으로 저장합니다.
- `supplier` 필드에는 수신처 값이 저장됩니다.
- 신규 품목의 관리번호(`no`)는 현재 가장 큰 번호 + 1로 자동 생성됩니다.

예외:

- 공통 품목을 선택하지 않으면 `공통 품목을 선택해 주세요.`가 표시됩니다.
- 납품처를 직접 타이핑만 하고 목록에서 선택하지 않으면 `거래처를 선택해 주세요.`가 표시됩니다.
- 품목명이 없으면 `거래처별 품목명을 입력해 주세요.`가 표시됩니다.
- DB의 품목 기본키 충돌이 발생하면 다음 ID를 계산해 재시도합니다.

## 8.3 품목 단가 직접 저장

납품처별 품목 목록에서는 입고단가와 판매단가를 바로 수정해 저장할 수 있습니다.

중요:

- 이 변경은 `products` 테이블의 기준 단가를 바꿉니다.
- 이미 작성된 문서 품목의 단가는 자동으로 바뀌지 않습니다.
- 앞으로 문서 작성에서 해당 품목을 선택할 때 새 기준 단가가 사용됩니다.

## 8.4 단가 수정 탭

단가 수정 탭은 이미 발행된 문서 품목의 단가를 일괄 변경하는 기능입니다.

### 검색 조건

필수:

- 시작일
- 종료일
- 품목 선택 또는 품목명 입력

검증:

- 시작일/종료일이 없으면 검색할 수 없습니다.
- 시작일이 종료일보다 늦으면 검색할 수 없습니다.
- 품목명 또는 품목 선택이 없으면 검색할 수 없습니다.

### 직접입력 품목 검색

- 품목 선택에서 `직접입력`을 선택하면 등록 품목이 아닌 문서 품목을 검색합니다.
- 직접입력 상태에서 품목명을 입력하면 해당 키워드가 포함된 직접입력 품목을 찾습니다.
- 품목명이 비어 있으면 기간 내 직접입력 품목 전체가 대상입니다.

### 미리보기

검색 결과에는 다음 정보가 표시됩니다.

- 기준일
- 발급번호
- 납품처
- 수신처
- 품목명
- 수량
- 현재 입고단가
- 현재 판매단가
- 공급가액

적용 전 반드시 체크해야 하는 이유:

- 검색 결과 전체가 자동 적용되는 것이 아닙니다.
- 사용자가 체크한 항목만 변경 대상이 됩니다.

### 변경 적용

적용 조건:

- 변경할 입고단가 또는 판매단가 중 하나 이상 입력해야 합니다.
- 변경 대상 품목을 1개 이상 체크해야 합니다.

확인창 내용:

- 선택한 품목 수
- 영향 문서 수
- `products 테이블은 변경하지 않습니다.`

중요:

- 단가 수정 탭은 이미 발행된 `document_items`의 단가를 바꿉니다.
- 품목 마스터(`products`) 기준 단가는 변경하지 않습니다.
- 변경 후 이력이 `price_change_logs`에 남습니다.

예외:

- 변경 이력 테이블이 아직 없거나 스키마 캐시에 없으면 이력 목록은 빈 목록으로 처리될 수 있습니다.

## 9. 공급자 관리

공급자 정보는 문서의 공급자 영역에 들어갑니다.

문서 작성 화면 동작:

- 화면 진입 시 활성 공급자 목록을 불러옵니다.
- ID가 `1`인 활성 공급자가 있으면 기본 공급자로 자동 입력합니다.
- 공급자 정보가 이미 입력되어 있으면 기본 공급자로 덮어쓰지 않습니다.

공급자 불러오기:

- 상호, 등록번호, 성명, 주소, 업태, 종목으로 검색할 수 있습니다.
- 공급자를 더블클릭하면 문서의 공급자 정보가 교체됩니다.

예외:

- 공급자 정보는 문서 저장 시 문서에 복사되어 저장됩니다.
- 나중에 공급자 마스터를 수정해도 이미 저장된 문서의 공급자 정보는 자동 변경되지 않습니다.

## 10. 월마감

월마감은 발행이력의 문서 품목을 월별, 납품처별로 집계합니다.

## 10.1 집계 기준

대상:

- 삭제되지 않은 문서
- 문서 상태가 `ST01`이 아닌 문서
- 삭제되지 않은 문서 품목
- 품목 상태가 `ST01`이 아닌 품목

날짜 기준:

1. 품목 입고일
2. 문서 입고일
3. 품목 발주일
4. 문서 발주일

위 순서로 먼저 있는 날짜를 기준일로 사용합니다.

월 필터:

- 기준일의 `YYYY-MM`이 선택한 연월과 같아야 합니다.

납품처 필터:

- 문서의 `clientId`가 선택한 납품처 ID와 같아야 합니다.

## 10.2 품목명 기준

월마감 품목명은 다음 우선순위로 결정됩니다.

1. 문서 품목에 `productId`가 있고, 현재 품목 마스터에서 찾을 수 있으면 품목 마스터의 출고의뢰서 품목명
2. 그렇지 않으면 문서 품목에 저장된 품목명

이 뜻:

- 등록 품목은 현재 품목명 기준으로 보일 수 있습니다.
- 직접입력 품목이나 삭제된 품목은 문서에 저장된 품목명을 사용합니다.

## 10.3 금액 계산

- 공급가액은 문서 품목에 저장된 `supply` 값을 사용합니다.
- VAT 체크가 켜져 있으면 `supply × 10%`를 반올림합니다.
- 합계는 공급가액 + 부가세입니다.

주의:

- 문서 작성 당시 저장된 공급가액이 기준입니다.
- 품목 마스터의 단가만 수정해도 기존 월마감 금액은 바뀌지 않습니다.
- 단가 수정 탭으로 문서 품목 단가를 바꾸면 월마감에 반영됩니다.

## 10.4 비고 저장

- 각 월마감 행의 비고는 `document_items.monthly_closing_note`에 저장됩니다.
- 비고 입력 후 Enter를 누르면 저장합니다.
- 저장 대상 품목이 없거나 삭제된 경우 `비고를 저장할 대상 품목을 찾지 못했습니다.`가 발생합니다.

## 10.5 종합장

종합장은 선택한 월의 납품처별 총 공급가액을 보여줍니다.

집계 기준:

- 문서 상태와 품목 상태가 정상인 항목만 포함합니다.
- 기준일이 선택한 월에 속한 품목만 포함합니다.
- 금액은 `supply` 합계입니다.

비고:

- 종합장 비고는 월별, 납품처별로 별도 저장됩니다.
- 입력 후 포커스를 벗어나거나 Enter를 누르면 저장됩니다.

## 11. 일일판매

일일판매는 관리자 전용 화면입니다.

## 11.1 행 생성 기준

일일판매는 품목 마스터와 문서 품목을 함께 사용합니다.

먼저:

- 삭제되지 않은 모든 품목 마스터 기준으로 행을 만듭니다.
- 이 때문에 해당 월 판매가 없어도 품목 행이 보일 수 있습니다.

그다음:

- 선택 월에 해당하는 문서 품목 수량을 일자별로 더합니다.

## 11.2 집계 제외 기준

다음은 제외됩니다.

- 문서 전체가 거래취소(`ST01`)인 경우
- 문서 품목이 거래취소(`ST01`)인 경우
- 기준일이 없는 품목
- 기준일의 연월이 선택 월과 다른 품목

기준일 우선순위:

1. 품목 입고일
2. 문서 입고일
3. 품목 발주일
4. 문서 발주일

## 11.3 행 묶음 기준

행을 구분하는 키:

- 납품처 ID 또는 납품처명
- 품목 ID 또는 품목명
- 수신처

등록 품목:

- 품목 ID 기준으로 묶입니다.

직접입력 품목:

- 품목 ID가 없으므로 품목명과 납품처명, 수신처 기준으로 묶입니다.

## 11.4 금액과 단가

일자별 수량:

- 기준일의 일자 위치에 수량을 더합니다.

입고금액:

- 문서 품목 입고단가 × 수량

출고금액:

- 문서 품목 판매단가 × 수량

가중 평균 단가:

- 월 수량이 있으면 `총 금액 ÷ 총 수량`으로 계산합니다.
- 월 수량이 없으면 품목 마스터의 기준 단가를 표시합니다.

## 11.5 비고 저장

- 행별 비고는 선택 연월과 행 키 기준으로 저장됩니다.
- 입력 후 Enter 또는 포커스 이동 시 저장됩니다.

## 12. 계정 관리

계정 관리는 관리자 전용입니다.

## 12.1 계정 추가/수정

필수:

- 아이디
- 비밀번호
- 이름

수정 시:

- 새 비밀번호를 비워두면 기존 비밀번호를 유지합니다.
- 새 비밀번호를 입력하면 해시 후 저장합니다.

권한:

- `user`: 일반 사용자
- `admin`: 관리자

## 12.2 비밀번호 처리

- 신규 계정 비밀번호는 해시로 저장됩니다.
- 기존 평문 비밀번호 계정은 로그인 성공 시 현재 알고리즘의 해시 비밀번호로 전환됩니다.
- 비밀번호 초기화 시 새 비밀번호를 해시로 저장하고 임시 비밀번호 상태로 표시합니다.

## 12.3 로그인 실패와 잠금

- 비밀번호가 틀리면 로그인 실패 횟수가 증가합니다.
- 실패 횟수가 5회 이상이면 계정이 잠깁니다.
- 잠긴 계정은 로그인할 수 없고 관리자 조치가 필요합니다.

## 13. 보도자료

보도자료는 업무 참고용 자료실입니다.

일반 흐름:

- 목록에서 보도자료를 조회합니다.
- 항목을 클릭하면 상세 화면으로 이동합니다.
- 보도자료는 문서 작성, 수주대장, 월마감 집계에는 직접 영향을 주지 않습니다.

## 14. 자주 발생하는 상황별 처리

### 14.1 품목 목록이 안 나올 때

확인 순서:

1. 납품처를 목록에서 선택했는지 확인합니다.
2. 수신처를 입력했는지 확인합니다.
3. 품목 관리에서 해당 납품처와 수신처로 등록된 품목이 있는지 확인합니다.
4. 품목이 비활성/삭제 상태가 아닌지 확인합니다.

임시 처리:

- 급한 경우 문서 작성에서 `직접입력`으로 품목을 입력할 수 있습니다.
- 단, 직접입력 품목은 품목 마스터 ID가 없으므로 이후 단가/포장 기준 자동화가 제한됩니다.

### 14.2 저장 버튼을 눌러도 저장이 안 될 때

확인할 것:

- 발주일, 입고일이 있는지
- 납품처를 목록에서 선택했는지
- 수신처가 있는지
- 납품주소가 있는지
- 품목명이 있는 품목이 1개 이상 있는지

특히 납품처:

- 화면에 납품처명이 보여도 직접 타이핑만 했다면 저장되지 않습니다.
- 반드시 드롭다운 목록에서 선택해야 합니다.

### 14.3 월마감 금액이 예상과 다를 때

확인 순서:

1. 문서 또는 품목이 거래취소 상태인지 확인합니다.
2. 품목 입고일과 문서 입고일이 선택 월에 속하는지 확인합니다.
3. 문서 품목의 공급가액이 직접 수정되었는지 확인합니다.
4. VAT 체크 여부를 확인합니다.
5. 단가 수정 탭으로 과거 문서 품목 단가가 변경되었는지 확인합니다.

### 14.4 수주대장에 문서 품목이 안 맞을 때

가능한 원인:

- 문서 상세에서 품목을 삭제/추가했지만 수정 저장이 실패했습니다.
- 과거 수주대장 데이터에 `document_item_id`가 없어 안전 동기화가 막혔습니다.
- 문서 전체 거래취소 또는 품목취소가 수주대장 상태에 반영되어 있습니다.

처리:

- 발행 이력 상세에서 다시 수정 저장을 시도합니다.
- 백필 SQL 안내 오류가 나오면 운영자가 `order_book document_item_id` 백필 SQL을 먼저 실행해야 합니다.

### 14.5 품목 단가를 바꿨는데 기존 문서가 안 바뀔 때

정상 동작입니다.

- 품목 관리 목록에서 단가를 저장하면 `products` 기준 단가만 바뀝니다.
- 기존 발행 문서의 품목 단가는 보존됩니다.
- 기존 문서 단가를 바꾸려면 `품목 관리 > 단가 수정` 탭을 사용해야 합니다.

### 14.6 거래취소와 출고상태의 차이

거래취소:

- 거래 자체를 집계에서 제외하는 상태입니다.
- 문서, 문서 품목, 수주대장 상태가 `ST01`이 됩니다.

출고상태:

- 물류 처리 상태입니다.
- 값은 `미출고` 또는 `출고`입니다.
- 거래취소와 별개로 관리됩니다.

운영 주의:

- 거래취소 항목을 출고 처리하지 않도록 목록에서 상태를 확인해야 합니다.
